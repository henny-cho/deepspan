# deepspan Stable API Reference

This document defines the stability guarantees for deepspan interfaces, organized
into three tiers. HWIP plugin authors should only depend on **Tier 1** interfaces.

---

## Tier 1 — Stable (SemVer-protected, no breaking changes without major version bump)

### Go interfaces

| Symbol | Package | Notes |
|---|---|---|
| `Submitter` | `github.com/myorg/deepspan/l4-server/pkg/hwip` | Core HWIP command interface |
| `SubmitterInfo` | `github.com/myorg/deepspan/l4-server/pkg/hwip` | Optional metadata interface |
| `SubmitterFactory` | `github.com/myorg/deepspan/l4-server/pkg/hwip` | Factory interface |
| `Register()` | `github.com/myorg/deepspan/l4-server/pkg/hwip` | Plugin registration |
| `NewServiceFromRegistry()` | `github.com/myorg/deepspan/l4-server/pkg/hwip` | Service construction |

### CMake targets (via `find_package(DeepspanPlatform)`)

| Target | Header | Notes |
|---|---|---|
| `Deepspan::deepspan-appframework` | `deepspan/appframework.hpp` | Session manager, device pool, circuit breaker |
| `Deepspan::deepspan-userlib` | `deepspan/userlib.hpp` | C++23 async ioctl/io_uring client |

### CMake functions (via `include(DeepspanHwip)`)

| Function | Notes |
|---|---|
| `deepspan_hwip_target()` | Creates an HWIP shared library target |
| `deepspan_hwip_codegen()` | Adds codegen custom target from hwip.yaml |

### Protobuf / ConnectRPC

| Symbol | File | Notes |
|---|---|---|
| `deepspan.v1.HwipService` | `l5-proto/deepspan/v1/device.proto` | Platform-level RPC service |
| `deepspan.v1.DeviceInfo` | same | Device metadata message |
| `deepspan.v1.DeviceState` | same | Device state enum |

### Linux UAPI

| Header | Notes |
|---|---|
| `linux/deepspan.h` | Base HWIP ioctl definitions (URING_CMD interface) |

### Python SDK

| Symbol | Module | Notes |
|---|---|---|
| `DeepspanClient` | `deepspan.client` | Synchronous ConnectRPC client |
| `HwipExtension` | `deepspan.client` | Protocol for HWIP extension objects |

## Tier 2 — Deprecated (will emit warnings before removal)

These symbols exist today but will be removed in a future major version. Prefer
Tier 1 alternatives.

| Symbol | Tier 1 replacement | Removal milestone |
|---|---|---|
| `DeepspanClient.submit_request(opcode=int)` | Use typed HWIP extension (`l6-sdk`) | v1.0.0 |

---

## Tier 3 — Internal (no stability guarantee)

Anything under these paths may change or disappear without notice:

- `l4-server/internal/` — server implementation details
- `l2-firmware/lib/` — Zephyr firmware internals
- `l3-hw-model/internal/` — simulator internals
- `l4-mgmt-daemon/internal/` — daemon internals
- `l5-gen/` — generated files (regenerated by codegen pipeline)

---

## Versioning policy

deepspan follows [Semantic Versioning 2.0.0](https://semver.org/).

- **Patch** releases (0.x.**y**): bug fixes only, no API changes.
- **Minor** releases (0.**x**.0): new Tier 1 symbols added; existing symbols unchanged.
- **Major** releases (**x**.0.0): breaking changes allowed; Tier 2 symbols removed.

HWIP plugins should pin `deepspan` to a **minor** version range:
```
# west.yml
revision: v0.3.0   # SHA or tag — never branch: main
```
